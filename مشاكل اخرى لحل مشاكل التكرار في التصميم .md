1. CSS Tokens كاملة

هذا أول شيء.

لازم أي token تستخدمه Amana يكون موجوداً فعلاً:

--space-xs
--space-sm
--space-md
--space-lg
--space-xl
--space-2xl
--space-3xl
--space-4xl

--radius-sm
--radius-md
--radius-lg
--radius-xl
--radius-2xl

--shadow-sm
--shadow-md
--shadow-lg
--shadow-xl

--transition-fast
--transition-smooth

--text-xs
--text-sm
--text-md
--text-lg
--text-xl
--text-2xl
--text-3xl

--color-success
--color-warning
--color-danger
--border-color
--content-width
--readable-width
--wide-width

لأن Card(variant: glass) سيحتاج radius وshadow وsurface.
وlayout: bento سيحتاج spacing.
وhover: lift سيحتاج transition.

لو tokens ناقصة، كل شيء جديد سيطلع ناقص أو متشابه.

2. إصلاح radius scale

لازم large وxl لا يكونوا نفس القيمة.

مثلاً:

soft: ["10px", "16px", "22px", "28px", "36px"]

ثم:

--radius-small: 10px;
--radius-medium: 16px;
--radius-large: 22px;
--radius-xl: 28px;
--radius-2xl: 36px;

هذا مهم لأن variants مثل:

Card(variant: brutal)
Card(variant: luxury)
Card(variant: dashboard)

ستحتاج اختلاف واضح في الحواف.

3. إصلاح hover الحقيقي

لازم:

hover: lift

يولد:

.selector {
  transition: var(--transition-smooth);
}

.selector:hover {
  transform: translateY(-6px);
  box-shadow: var(--shadow-large);
}

وليس فقط transition بدون :hover.

هذه ضرورية قبل variants، لأن كل variants تقريباً ستحتاج hover مختلف.

4. Diagnostics بدل السكوت

أي property غير معروفة لازم تعطي خطأ أو warning.

مثلاً:

Card(variant: glsas)

لازم يعطي:

Unknown Card variant "glsas".
Did you mean "glass"?

ومثلاً:

section Hero:
    layout: bentoo

يعطي:

Unknown layout "bentoo".
Did you mean "bento"?

لو compiler يسكت، الذكاء الاصطناعي سيكتب أخطاء وانت لن تعرف لماذا التصميم لم يتغير.

5. إصلاح FormField وNavbar وIcon

هذه صغيرة لكنها مهمة:

FormField يحترم required
Navbar يدعم links أو children
Iconify class/icon تنصلح

لأنك عندما تبني components وvariants جديدة، لا تريد مكونات أساسية تتصرف غلط.

6. Tag allowlist

قبل custom components لازم تمنع توليد tags خطيرة.

يعني لا تسمح بـ:

script:
    "..."

ولا:

iframe:
    "..."

إلا إذا أضفت سياسة أمان واضحة لاحقاً.

لأن custom components ستفتح الباب للمطور يكتب HTML-like syntax، فلازم الأمان يسبقها.

متى تعتبر Foundation Gate انتهت فعلاً؟

لا تعتبرها انتهت فقط لأنك كتبت الكود. اعتبرها انتهت عندما يتحقق هذا:

1. CSS Tokens

أي token يرجعه parser أو codegen موجود فعلاً في CSS النهائي.

مثلاً لا يوجد شيء مثل:

padding: var(--space-2xl);

بدون أن يكون:

--space-2xl: 3rem;
2. Radius

لا يوجد تكرار غبي مثل:

--radius-large: 34px;
--radius-xl: 34px;

بل scale واضح:

--radius-sm: 10px;
--radius-md: 16px;
--radius-lg: 22px;
--radius-xl: 28px;
--radius-2xl: 36px;
3. Hover

لو كتبت:

hover: lift

يجب أن ترى في CSS:

.selector:hover {
  transform: translateY(-6px);
}

وليس فقط transition.

4. Diagnostics

أي شيء غلط لا يمر بصمت.

مثلاً:

Card(variant: glsas)

لازم يعطي warning أو error واضح:

Unknown Card variant "glsas". Did you mean "glass"?
5. FormField / Navbar / Icon

هذه تعتبر done عندما:

FormField(required: true) يولّد required و aria-required
Navbar يدعم links أو children
Icon(name: "...") يولّد icon صحيح و class صحيح
6. Tag allowlist

لو كتب أحد:

script:
    "alert(1)"

لازم compiler يمنعه أو يولّد diagnostic.
