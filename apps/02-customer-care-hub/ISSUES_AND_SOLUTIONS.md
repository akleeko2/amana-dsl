# وثيقة المشاكل والحلول التقنية (ISSUES_AND_SOLUTIONS.md)

توضح هذه الوثيقة جميع المشاكل التقنية والنحوية والتنسيقية التي واجهتنا أثناء تطوير تطبيق **Customer Care Hub (apps/02-customer-care-hub)** باستخدام لغة البرمجة **Amana DSL**، وكيف تم تشخيص كل مشكلة وحلها بشكل جذري لضمان ثبات النظام وتجاوبه الكامل مع جميع شاشات الأجهزة.

---

## 1. مشاكل ترميز الملفات (Encoding & UTF-8 BOM)

### المشكلة: خطأ "Unexpected character" عند الترجمة
- **التشخيص:** عند استخدام أوامر PowerShell مثل `Set-Content` أو إعادة التوجيه لحفظ وتعديل الملفات، يقوم نظام Windows تلقائياً بإضافة علامة ترتيب البايتات (BOM - Byte Order Mark) في بداية ملفات الـ UTF-8 (تظهر كـ `EF BB BF` بالبايت). مترجم Amana DSL يرفض هذه العلامة ويُظهر خطأ مثل:
  ```
  Unexpected character '﻿' (0xfeff)
  ```
- **الحل:**
  1. تجنب استخدام أوامر PowerShell القياسية لتعديل الملفات بشكل مباشر.
  2. استخدام كود Python أو كائنات دوت نت لتصدير الملفات بصيغة UTF-8 صافية بدون BOM:
     ```powershell
     $encNoBom = New-Object System.Text.UTF8Encoding($false)
     [System.IO.File]::WriteAllText($filePath, $content, $encNoBom)
     ```

### المشكلة: فقدان أول حرف بعد تنظيف ملفات الـ BOM
- **التشخيص:** محاولات تنظيف ملفات الـ BOM السابقة بواسطة كود يقرأ الملف كـ string ثم يعيد كتابته أدت في بعض الأحيان إلى حذف الحرف الأول الفعلي للملف (مثل تحويل `view` إلى `iew`) بسبب التعامل الخاطئ مع الإزاحة (offset).
- **الحل:** قراءة بايتات الملف raw أولاً، والتحقق يدوياً من وجود الثلاثة بايتات الخاصة بالـ BOM (`0xEF, 0xBB, 0xBF`) قبل إزالتها بدقة، ثم كتابة البايتات المتبقية بدون أي تلاعب بالـ string.

## 2. مشاكل قواعد وصياغة لغة أمانة (Amana DSL Syntax & Constraints)

### المشكلة: خطأ نحوي عند استخدام التجاوب المخصص في الـ Grid
- **التشخيص:** محاولة كتابة `mobile.columns: 1` أو خصائص فرعية مشابهة داخل كتلة التجاوب لـ `Grid` تؤدي لفشل المترجم.
- **الحل:** تدعم لغة Amana سمات التجاوب بأسلوب مباشر ومبسط مثل:
  ```amana
  Grid(columns: "3", responsive: "stacked"):
  ```
  أو استخدام فئات CSS المخصصة للتحكم الدقيق بأبعاد الشاشات المختلفة بدلاً من الاعتماد الكلي على خصائص الـ Grid المدمجة.

### المشكلة: خطأ في المكونات ذاتية الإغلاق (Self-Closing Components)
- **التشخيص:** كتابة نقطتين رأسيتين `:` بعد المكونات التي لا تحتوي على كتلة برمجية فرعية (مثال: `Navbar():`) تتسبب في فشل التحليل النحوي (Parser Error).
- **الحل:** إزالة النقطتين الرأسيتين تماماً لتصبح استدعاءات نظيفة مثل `Navbar()` عندما لا يتبعها عناصر أبناء.

### المشكلة: عدم دعم الـ f-strings في مسارات التوجيه (Redirect Paths)
- **التشخيص:** عند استخدام التوجيه الشرطي بعد نجاح العمليات، فإن كتابة مسارات ديناميكية باستخدام صيغة الـ f-string مثل `redirect success -> f"/tickets/{id}"` تفشل في الترجمة.
- **الحل:** استخدام مسارات ثابتة وواضحة مثل `redirect success -> /tickets` أو إدارة التوجيه عبر المعاملات المدعومة.

### المشكلة: استخدام الشرطة الواصلة في مفاتيح التوكينز (Tokens Hyphens)
- **التشخيص:** تعريف مفاتيح التوكينز باستخدام الشرطة الواصلة مثل `brand-soft` يسبب مشاكل للمحلل النحوي للغة.
- **الحل:** استخدام الشرطة السفلية (Underscore) دائماً لتسمية المفاتيح مثل `brand_soft`.

### المشكلة: استخدام وسوم `script` أو `style` داخل كتلة الـ `render`
- **التشخيص:** يرفض المترجم تماماً إدراج وسوم HTML التقليدية مثل `<style>` أو `<script>` داخل كتلة الـ `render:` الخاصة بالصفحة.
- **الحل:** كتابة كتل الـ `style:` المخصصة في المستوى الخارجي للصفحة (خارج كتلة الـ `render:`).

### المشكلة: استخدام السمة `:style` الديناميكية
- **التشخيص:** محاولة كتابة `:style="..."` كسمة للعناصر لتمرير تنسيقات ديناميكية غير مدعوم في شجرة الـ render للغة.
- **الحل:** الاعتماد الكامل على فئات CSS وتوكينات التصميم، واستخدام حقول البيانات أو الـ Alpine.js للتحكم بالأنماط الديناميكية بشكل غير مباشر.

### المشكلة: غياب ميزة الاتصال الشبكي الخارجي `network.outbound`
- **التشخيص:** وجود `capability network.outbound` في ملف `app.amana` يُسبب خطأ ترجمة لعدم دعم هذه الإمكانية في النسخة الحالية من الإطار.
- **الحل:** إزالتها تماماً من ملف الإعدادات `app.amana`.

## 3. مشاكل التنسيق والتجاوب وارتفاع الشاشة (Layout & CSS)

### المشكلة: تداخل Grid الإطار البرمجي مع عرض الصفحة الرئيسي
- **التشخيص:** يفرض إطار Amana تنسيق Grid تلقائي على الحاوية الخارجية الأولى للمكونات عبر الأنماط الافتراضية:
  ```css
  .amana-runtime-shell > :not(script):not(style) {
      display: grid;
      grid-template-columns: minmax(14rem, 18rem) minmax(0, 1fr);
  }
  ```
  هذا التنسيق كان يؤدي لتدمير تنسيق الهيكل المخصص (App Shell) في الأجهزة الذكية والأبعاد المختلفة، وجعل عرض المحتوى الرئيسي ينضغط ليصبح 0.
- **الحل:** إبطال مفعول الـ Grid الافتراضي وجعل الحاوية تتصرف ككتلة كاملة، وتطبيق التنسيق على الـ App Shell مباشرة:
  ```css
  .amana-runtime-shell {
      display: block !important;
  }
  .app-shell {
      grid-column: 1 / -1;
      width: 100%;
  }
  ```

### المشكلة: تعطل شريط التمرير (Scroll) بالكامل وقطع المحتوى في الأجزاء السفلية
- **التشخيص:** يقوم مترجم Amana DSL تلقائياً بإنشاء عنصر `div` بدون كلاس (Wrapper) لتغليف كتلة الـ `x-data` الخاصة بـ Alpine.js مباشرة تحت الحاوية الرئيسية `.amana-runtime-shell`. وبسبب عدم تعيين ارتفاع محدد لهذا العنصر الوسيط (Wrapper Div)، تتمدد الصفحة طولياً بمقدار محتواها متجاهلةً التقييد بالكامل، وبما أن الإطار يفرض `overflow: hidden` على الحاويات الأعلى، يتعطل الـ Scroll وتُقص الحواف السفلية للصفحات دون إمكانية للوصول إليها.
- **الحل:** استهداف عنصر الـ Wrapper الوسيط بشكل مباشر وفرض قيود الارتفاع والتمرير المرن (Flex):
  ```css
  .amana-runtime-shell > div:
      display: "flex"
      flex-direction: "column"
      height: "100%"
      min-height: "0"
      width: "100%"
      overflow: "hidden"
  ```
  مع تطبيق قيم `height: 100%` و `overflow: hidden` على `.app-shell` وتعيين `overflow-y: auto` و `min-height: 0` على الـ `.dashboard-main`.

### المشكلة: تمدد مكونات الـ Grid رأسياً بشكل مشوه
- **التشخيص:** عند استخدام أبعاد مختلفة للشاشات، تتمدد بطاقات الـ Grid (مثل الـ KPIs وجداول البيانات) رأسياً لتملأ كامل طول الصف بالتساوي، مما يؤدي لشكل غير متناسق وفراغات كبيرة.
- **الحل:** ضبط محاذاة العناصر رأسياً على حاويات الـ Grid لتكون في البداية بدلاً من التمدد الافتراضي:
  ```css
  .dashboard-grid, .kpi-row, .reports-grid {
      align-items: start;
  }
  ```

### المشكلة: تحول النصوص إلى أحرف رأسية في الشاشات الصغيرة
- **التشخيص:** في الشاشات ذات العرض المحدود جداً، وفي حال عدم تحديد نمط كسر النصوص، يضغط المتصفح النصوص الطويلة (مثل عناوين البطاقات) لتصبح عمودية حرفاً بحرف لعدم وجود مساحة كافية.
- **الحل:** فرض نمط الالتفاف التلقائي وكسر الكلمات الطويلة بمرونة:
  ```css
  .dash-title, .card-title {
      overflow-wrap: break-word;
      word-break: break-word;
      max-width: 100%;
  }
  ```

## 4. مشاكل البيانات والتهيئة (Database & Seeds)

### المشكلة: فشل تهيئة قاعدة البيانات بسبب حقول مفقودة
- **التشخيص:** عند تشغيل التهيئة الأولية (Seed Database)، يفشل نموذج `KnowledgeArticle` لوجود حقل مطلوب باسم `content` لم يتم تمريره في البيانات المبدئية.
- **الحل:** تحديث ملفات الـ seeds لتشمل جميع الحقول الإلزامية المسجلة في تعريف النموذج.

### المشكلة: نقص معرف الوكيل (Agent ID) في نماذج التعديل
- **التشخيص:** فشل عمليات التحديث الخاصة بالوكلاء والعملاء لعدم إدراج حقل معرف السجل المخفي `id` كمدخل أساسي داخل نموذج الإرسال (Form).
- **الحل:** إضافة عنصر إدخال مخفي `<input type="hidden" name="id" ...>` لضمان تمرير المعرف في طلبات `POST` الموجهة للتحديث.
